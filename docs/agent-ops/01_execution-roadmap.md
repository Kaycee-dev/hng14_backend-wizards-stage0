# Execution Roadmap

## Combined Order

1. Governance bootstrap
   - inspect target repo
   - choose stack and route owner
   - create sprint artifacts
2. Endpoint implementation
   - input validation
   - Genderize integration
   - response transform
3. Hardening
   - error mapping
   - tests
   - repeated-request stability
4. Deployment
   - accepted platform
   - public URL
   - live CORS verification
5. Submission
   - README
   - sign-off log
   - submission fields

## Dependency Rules

- route ownership must be settled before implementation
- correctness precedes performance tuning
- local verification precedes deployment
- live verification precedes submission
- proof updates happen in the same session as the code or deployment work
