# Session and Gates

## Session Start

Before major work:

1. read the current trackers and logs
2. read `docs/agent-ops/04_deadline-overlay.md` and `docs/agent-ops/06_delivery-state.md`
3. confirm the day/date, target gate, blockers, and next actions
4. confirm prerequisite work is complete

## Same-Session Logging

After substantive work:

- update the relevant progress tracker
- append evidence
- append decisions for non-trivial choices
- update the proof journal
- update `docs/agent-ops/06_delivery-state.md`

## Gates

- Gate A: route ownership and local contract shell
- Gate B: local integration and verification
- Gate C: public deployment and live CORS proof
- Gate D: final live smoke and submission readiness
