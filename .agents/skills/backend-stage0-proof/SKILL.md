# Backend Stage0 Proof

Use this skill to keep the evidence trail, README, and submission material defensible.

## Start Here

1. Read `backend-wizards-stage0-proof-sprint_v1_0.md`.
2. Read the active progress, evidence, and decisions logs in the sprint, delivery, and proof folders.
3. Read `docs/backend-stage0-sprint/10_signoff_log.md`.
4. Read `docs/backend-stage0-delivery/01_submission_checklist.md`.

## Workflow

1. Capture the exact commands, files, URLs, and live behaviors that prove the work.
2. Keep the README aligned with the code and deployed endpoint.
3. Update the sign-off log and submission checklist before final submission.
4. Leave the next exact starting point in the delivery state and journal.

## Hard Rules

- Never invent a live URL, test pass, or submission confirmation.
- Do not let the README claim behavior that the live endpoint does not currently exhibit.
- Keep stack wording consistent between README, sign-off log, and submission checklist.
- Record ambiguity decisions instead of flattening them away.
